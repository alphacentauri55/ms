Estructura superior:

!DOCTYPE html {Le dice al navegador que el documento es de tipo HTML y asi lo puede identificar}

html lang="en" {Le indica al navegador el lenguaje de la pagina: 'en', 'es', etc...}
-La etiqueta html tiene etiqueta de cierre, adentro va el head y el body


Head:

head

meta charset="UTF-8" {Permite que el navegador acepte caracteres especiales como tildes o emojis}

meta http-equiv="X-UA-Compatible" content="IE=edge" {Muestra los sitios habilitados para Google Chrome Frame, permite las funciones de rendimiento y seguridad de Google Chrome}

meta name="viewport" content="width=device-width, initial-scale=1.0" { hace que la pagina se adapte a la anchura del dispositivo que se esté usando, como en celulares}

meta name="robots" content="index,follow" {hace que permitamos al navegador usar robots para ubicar la página y darle visibilidad dependiendo de la búsqueda, si no se le quiere dar permiso se pone “unfollow” en ves de follow}

meta name="description" content="texto" {Esto es la descripción que aparece abajo de la url cuando busco una página en Google o en cualquier navegador}

title Nombre o titulo de la pagina title {tituló que aparece en la parte de arriba de la pestaña en el navegador}

link rel="stylesheet" href="" {Sirve para poner los links de las paginas de estilos o para incluir una fuente externa}

Imagenes y videos


Imagenes

-Tamaño máximo recomendado para una imágen: 70kb

-Es bueno ponerle un title='' a las imagenes o en general a cualquier vinculo

figure

....img src="./" alt="Imagen de..."
....figcaption descripcion figcaption

figure

-Figure crea un contenedor a la imagen, es mejor que usar un div por semantica y buenas practicas
-figcaption pone una descripcion de la imagen, como para dar derechos de autor o pequeños titulos

Herramientas para optimizar imágenes:

-Tiny PNG: Comprime el tamaño de una imagen,para hacerla más ligera.

-Verefix: Elimina los metadatos de una imagen, para reducir su tamaño.(como la localización o la hora a la cual fue tomada la foto)


Videos:

-El atributo controls permite poner controles de pausa, sonido, etc...

-preload='auto' permite que el navegador renderice el video desde que lee la etiqueta, asi se descarga desde antes y el video se va a reproducir mas rapido

-En el src despues de la extension se puede poner "#t=10,60", esto indica que el video va a empezar en el segundo 10 cuando el usuario presione play y se pausara en el segundo 60 automaticamente

-Se le puede poner varias etiqueta 'source' para que el video tenga varias extensiones y asi pueda funcionar y el navegador pueda escoger el que mejor use, Esto para asegurar que cualquier navegador pueda mostrar el video.


video controls preload='auto'

source src="./video.mp4#t=10,60"
source src="./video.mp5"
source src="./video.mp6"

video

Nota:

-poner a reproducir el video automaticamente en la pagina sin que el usuario lo decida es una mala practica y es penalizable en algunos navegadores (esto no tiene nada que ver con los atributos de arriba, #t no reproduce el video sin que el usuario le de play)

Modelo BEM


Block__Element--Modifier

Block:
-Un bloque es una etiqueta que es independiente, no necesita de nadie para existir: Header, nav, footer, formulario, container, etc...
-Se nombran con el mismo nombre de la etiqueta: header class='header'

Element:
-Un elemento dpende directamente de un bloque para existir, es dependiente del bloque y esta ligado a este
-Se nombra con el nombre del bloque__elemento: input class='form__input'

Modifier:
-Puede ser un bloque o un elemento, indican una modificacion a la estrucctura, cuando una etiqueta tiene una modificacion diferente a la de sus hermanos.
-Se nombra con -- despues del elemento: li class=' nav__item nav__item--blue'


Emmet


-Emmet es un sistema de abreviacion de etiquetas que permite crear estructuras en html de forma mas rapida

Ejemplos:

-etiqueta.algo: input.nombre = input class='nombre'

-etiqueta#algo: input#nombre = input id='nombre'

div>p: Pone un p dentro del div

div+p: pone el p despues de la etiqueta div

div>li*5: Crea 5 li dentro del div

-2: Utilizando las etiquetas style style en el head. No es una buena practica cuando son muchas lineas de codigo, solo cuando son algunas pocas debido a que tu pagina se demoraria mucho en cargar mientras lee todas las lineas

-3: Como atributo de la etiqueta: es una mala practica y hay que evitarlo lo mayor posible porque esto puede reescribir algunas lineas que esten en el archivo de stilos y generar problemas

Especificidad


Al usar css se tiene un orden en la forma de leer un estilo:

-1: Primero se toma en cuenta la importancia, atributo !important o estilos del navegador
-2: Segundo se toma en cuenta la especificidad
-3: Por ultimo se toma el orden en las fuentes {si una etiqueta tiene 2 atributos con diferentes medidas se toma en cuenta el que este mas abajo en el documento}


-La especificidad es la manera mediante la cual los navegadores deciden qué valores de una propiedad CSS son más relevantes para un elemento y por lo tanto la aplican

Selectoores y su especificdad:

!important: 5

inline styles: 4

#id: 3

.class: 2

tag: 1

Ejemplo:

h1 class='title' id='titulo'
....Titulo

color: blue;
}

#titulo {
color: red;
}

-Al inspeccionar codigo en la pagina web, me muestra los estilos de mas importantes a menos importantes, asi se a cual se le esta dando prioridad o porque hay errores


Medidas, herencia y combinadores


Tipos de medidas:

Medidas absolutas
No cambia el tamaño respecto a la pantalla que se use o al tamaño de esta:

mm = milimetros
cm = centimetros
in = pulgada
pc = picas
px = pixeles

Medidas relativas
Estas medidas heredan el tamaño o se basan en algun tamaño que se alla seleccionado, el valor ira cambiando segun la pagina cambie de tamaño:

% = porcentaje
em = em
rem = root em
vw = viewport width
vh = viewport height

1rem = 16px, esto porque rem coge el valor de la ruta que estemos usando y el navegador siempre le va a dar un valor de 16px al texto en html, almenos que se modifique con el selector de html en la hoja de estilos

Agregar valores:
margin: 5px 0 5px 0;

-Con 4 argumentos es : top, right, bottom, left -Con 3 argumentos es: top, right & left, bottom

-Con 2 argumentos es: top & bottom, right & left

Nota:
-Hay una forma de hacer que CSS calcule el tamaño de un elemento restándole cierta cantidad.

Ejemplo: colocar 2 cajas dentro de un div y quieres que cada una tome el 50% de ancho, pero que cada una tenga un margen de 10px.

.caja-hijo {
width: calc(50% - 20px);
}


Herencia:

Las etiquetas pueden heredar estilos de sus padres dependiendo de la etiqueta, hay algunas etiquetas que tienen sus propios estilos y por lo tanto aveces la herencia no funciona. Si yo aplico un tamaño de letra a un contenedor con un h1 y un p, solo se le aplicara al parrafo porque h1 tiene unos estilos por defecto

-Para aplicarle el estilo del contenedor se usa: h1 {font-size: inherit;} y asi hereda el valor de su padre


Combinadores:

-Hermano adyacente: h2 + p {
Toda etiuqueta p que este cerca de un h2 aplicale estos estilos, de arriba hacia abajo en el html y tiene que estar el h2 encima del p, no puede haber otra etiqueta entre ellas
}

-General sibling: h2 ~ p {
Toda etiqueta p que sea hermano general de h2 aplicale estos estilos, aqui lo unico que importa es si estan en el mismo contenedor de caja (en un div, un formulario, etc) y no importa si hay otras lineas o etiquetas que los separen con tal de estar en el mismo contenedor
}

-Child: div > p {
Toda etiqueta p que sea hijo de un div aplicale estos estilos, aqui se va a aplicar a todos los p que esten dentro de un div, tiene que ser espcificamente el padre directo, no puede estar detro del div y despues dentro de otra etiqueta
}

desendencia: div p {
todas las etiquetas p que esten adentro de un div sin importas en que otras etiquetas esten o cual sea su padre directo, es aplicar estilo a un hijo sin importar que mas halla, solo importa que este el p en alguna parte de un contenedor div
}

Pseudoelementos


Position


Static:
-Posicion estatica, la que esta por defecto

Relative:
-Posicion relativa, se el agrega top o bottom o left o right para asi mover el elemeneto de manera relativa

Absolute:
-Absolute hace que el elemento es removido del flujo del documento, se le agrega bottom o top o right o left para asi especificar en que parte quiero el elemento, si no se le pone estos atributos el elemento se va a sobreponer a sus hermanos, asi podria ponerse encima del hermano de abajo debido a que este ya se sale del flujo del documento y se sobrepone


-La diferencia entre absolute y relative es que el bottom, top, left y right del absolute es respecto a la pagina, en relative es respecto a su etiqueta, ejemplo: el bottom de absolute es el fondo de la pagina, el de relative es la parte de abajo de su propio contenedor, el bottom border.


Display


Block:
-Este hace que los elementos ocupen el 100% del width de la caja aun asi el contenido sea pequeño, este es el valor por defecto del display en las etiquetas div y p. Al ocupar el 100% del width hace que los elementos se pongan de forma vertical ya que estan uno debajo del otro

Inline:
-hace que los elementos se pongan uno al lado del otro si hay espacio, cuando la pagina o la seccion no tenga mas espacios estos se pondran abajo nuevamente uno al lado del otro. Cuando se usa este aributo no se puede usar margin o padding en la parte de arriba o abajo, tampoco usar width o height, no va a poder crecer porque esto es de forma lineal por lo tanto el codigo ignorara estos atributos

Inline-block:
-Es una fusion de los valores block e inline, se ocupa el 100% del width del contenido y si hay espacio permite que otros hermanos se pongan al lado, permite manipular el width y el height o tambien el padding o el margin


Dislplay: flex;

-Flex hace lo mismo que display inline-block pero se lo aplica a sus hijos contenedores, hace que sean flexibles y les cambia el display por defecto que traen, coloca a sus hijos en la direccion de row por defecto y se puede cambiar con el atributo de flex-direction: row/column/etc

flex-wrap: wrap; {hace que cuando la pantalla se encoja, los elementos pasen a la fila de abajo mediante se acabe el espacio, hace que sean flexibles al reacomodarse}

Justify-content: center; {Centra los elementos en la misma direccion del flex}

Align-items: center; {centra los elementos en la direccion opuesta a la direcion del flex}

flex-basis: 5rem; {especifíca la base flexible, el tamaño inicial de un elemento flexible}

order: 1; {Define el order para mostrar los elementos o contenedores hermanos, si a alguno no se le asigno un numero de order ent estos estaran al inicio en la izquierda y despues apareceran los que si tienen un order acorde a su numero}

flex-grow: 1; {hace que este contenedor rellene el espacio sobrante del contenedor padre, se expandira en su ancho o largo en la misma direccion del flex (si esta horizonotal, se expandira horizontal). El numero representa la importancia de ocupar el espacio, mientras mas grande sea el numero mas espacio ocupara rellenando el espacio sobrante del contenedor padre (si hay uno con flex-grow:2; este va a rellenar 2 tercios del espacio sobrante del contenedor padre y el que tenga flex-grow: 1; va a rellenar 1 tercio, y asi susesibamente)}

Usar el espacio del contenedor padre con los hijos:

Media quires y Responsive design


-Responsive design es la forma de adaptar el contenido de la pagina en diferentes tamaños y se hace aplicando estilos con media quires, hay muchos patrones para acomodar los elementos como: mostly fluid, layout shifter, column drop, etc...

-Los media quires son la forma de implementar estilo a nuestro codigo definiendo un tamaño de la pantalla, esto hace que el responsive design sea efectivo en todos los dispositivos

-Se recomienda usar minimo 3 archivos diferentes para cada enfoque: styles.css, tablet.css y desktop.css

-Los enlaces a los archivos con media quires deben de ir en la parte del head de menor a mayor (Primero el de mobile, despues tablet y abajo el de desktop)

-Mobile first: Siempre se tiene que diseñar primero en la pantalla de dispositivos moviles, de ahi ir ampliando hasta las pantallas mas grandes

-Una buena forma de implementar media quires es: Hacer de 300px-600px para mobile, despues para tablets uno de 600px-1000px, otro para computadores de 1000px-2000px y un ultimo para pantallas mas largas de +2000px

-Una buena pratica para que la pagina no se rompa con monitores muy grandes es poniendole un max-width: 3200px; a la etiqueta de html

implementar media quires:

Head
link rel="stylesheet" href="./css/styles.css"
link rel="stylesheet" href="./css/tablet.css" media="screen and (min-width: 600px)"
link rel="stylesheet" href="./css/desktop.css" media="screen and (min-width: 1000px)"
link rel="stylesheet" href="./css/max.css" media="screen and (min-width: 2000px)

Imagenes responsive:

Picture
source media="(min-width: 1300px)" srcset="./pics/grande.jpg"
source media="(min-width: 1000px)" srcset="./pics/mediano.jpg"
img src="./pics/pequeno.jpg" alt="Imagen pequeña de fuji"
Picture

-La etiqueta picture permite cargar imagenes responsive dependiendo del viewport, asi poder cargar la mejor imagen dependiendo del dispositivo y no cargar tanta informacion y peso de imagenes.

-La etiqueta source es para definir la imagen del viewport, es como si se remplazara el atributo src en la etiqueta de img, por eso solo se le pone css a la etiqueta img y se aplica a todas las imagenes dependiendo del tamaño de la pantalla

-Se debe poner desde la etiqueta con el viewport mas grande hasta la mas pequeña, es decir que se pone la etiqueta con la imagen mas grande y abajo la segunda mas grande y asi sucesivamente, si no no funciona

-Todo esto ayuda al rendimiento de la pagina para que la pagina no se quede cargando tanto ya que el navegador va a tener varias opciones acorde al tamaño del usuario y asi tener una mejor renderisacion

-En imagenes el width: 100%; es la mejor manera de responsive


Malas practicas


Utilizar tanto id en CSS

Utilizar el !important

Utilizar la etiqueta style dentro del archivo html

Utilizar el atributo style dentro de las etiquetas html

Utilizar div para contener todo ignorando los header, nav, section, article, etc.

No utilizar la etiqueta form para hacer formularios

Utilizar las etiquetas select y option para hacer selectores o menús desplegables.

No nombrar el primer archivo html del proyecto como index.html

No tener archivos .css para cada pantalla de un proyecto.

Tener todo el css junto en un solo archivo.

No ponerle el atributo alt a una imagen

Poner imágenes dentro de div en vez de figure

Utilizar textos solo en mayúscula en HTML, en vez de utilizar el atributo de CSS, text-transform, con el valor uppercase. Ya que al hacer esto pareciera que estuvieras gritando.

Poner videos que se reproduzcan solos.

No optimizar las imágenes.

No tener cuidado de cual es el formato ideal para las imágenes y su respectivo peso.

No tener cuidado con la respectiva semántica de HTML, y con la sintaxis adecuada para CSS.

No cerrar las etiquetas que se cierran en sí mismas como br

No comentar partes esenciales de tu código.

No poner la etiqueta meta name=”robots” en tu proyecto para que los navegadores los puedan ubicar mejor.

No usar la etiqueta meta name=”viewport” para hacer tu proyecto responsive.

No poner el atributo autocomplete=”” en los campos de tu formulario para hacerle la vida más fácil al usuario

No usar el atributo required en los campos obligatorios de tu formulario como una primera capa de seguridad
