Eliminar o cambiar cosas

git rm --cached archivo {quita el add del archivo, unstaged}

git rm --force {elimina los archivos del disco duro}

git reset hashCommit --hard {Elimina todos los cambios que se hicieron despues de este commit, se recrea la historia de los commits en git log}

git reset hashCommit --soft {Elimina el historial de los commits pero no los cambios}

git reset head archivo {Mueve el archivo a unstaged}

(git rm --cached elimina un archivo de staging. Es como si nunca le hubieses hecho git add a ese archivo. Mientras que git reset HEAD devuelve el archivo a su último commit, por si quiere modificar algo y luego le puedes hacer git add o git commit otra vez.)

git checkout hashCommit archivo {Me devuelvo a los cambios del archivo en ese commit}

git checkout master archivo {Me devuelvo al ultimo commit hecho, la ultima version}

(Si en vez de hacer esto hago un add y commit, se me guarda este archivo anterior como el ultimo cambio hecho, la ultima version seria esta)

git remote {Me muestra si hay algun origen añadido}

git remote -v {Me muestra la url de los origenes remotos}

git remote rm origin {Elimina el origin}

git remote set-url origin url {Cambie la url del origin, por ejemplo sirve cuando se cambia de HTTPS a SSH}


Interaccion remota y local


Gramatica:
git push origin master {Enviar al origen mi version del master}
git pull origin master {Traer del origen al master la ultima version}

git fetch {Traer datos de github a mi repositorio local}

git merge {Fucionar versiones de mi repositorio remoto al local}

git pull {Hace un fetch y un merge al mismo tiempo, trae la ultima version del repo remoto de github a mi repositorio local y lo fuciona en el directorio}

git push {Envia mi version del repo local al remoto de github, si es publico y soy colaborador se hace un merge automatico, de lo contrario se crearia un pull request}

git pull origin master --allow-unrelated-histories {Permite hacer un pull cuando las versiones o contenidos son algo distintos}

Rebase


Reescribe la historia del repositorio, cambia la historia de donde comenzó la rama y solo debe ser usado de manera local

-rebase es una mala práctica pero sirve para ocultar cambios en la historia

-Es cómo hacer un merge pero altera los cambios en los commits, sobrepone los cambios en la historia priorizando la rama en la cual se está y se hace el git rebase

Git rebase master {estando en la rama de cambios}
Git rebase ramaCambios {estando en el master}

Primero se hace el rebase en la rama de cambios y después en la master, después se puede eliminar la rama con git branch -d ramaCambios y así se cambia la historia como si nada hubiera pasado.


stash, clean, cherry-pick y amend


Stash:

git stash {Cuando hice algun cambio y no le quiero hacer un commit, esto hace que esos cambios se pongan stasheados temporalmente hasta que los quiera utilizar y hacer un commit, me devuelve a la historia del ultimo commit hecho}

-Sirve cuando quieres probar cosas pequeñas o no muy seguras las cuales no merecen una rama y asi tener estos cambios guardados temporalmente en caso de usarlos o no

git stash list {Muestra si hay algo stasheado 'WIP = Work In Progress' y me muestra el ultimo commit hecho de la version actual}

git stash pop {Me devuelve los cambios que tenia stasheados para hacerles un commit, se le quita el stash, esto debe hacerse en la misma rama que se hizo el git stash}

git stash drop {Elimina todos los cambios que esten en el stash, sirve cuado la cago y antes de hacer un commit puedo hacer un stash y despues un stash drop, asi el archivo se quedara en la version del ultimo commit antes de cagarla}

git stash branch nombreRama {Crea una rama donde trae los cambios que esten en el stash, asi puedo trabajar desde ahi y hacerle commit para experimentar y trabajar}


Clean:

git clean -f {Elimina archivos que no se le han hecho un add o un commit, archivos que estan por error como archivos de imagenes que no nesecitas}
(Si en el .gitignore hay hay estas extensiones de archivos, estas no se borraran con git clean porque git esta ignorando este tipo de archivos)

git clean --dry --run {Muestra los archivos que se eliminaran si se hace un git clean -f}

-A git solo le importa los archivos y no las carpetas, por eso con git clean solo se eliminan archivos, las carpetas deben ser eliminadas manualmente con comandos de linux.


Cherry-pick:

git cherry-pick hashCommit {Con el hash del commit, se trae ese cambio y lo incorpora el rama donde ejecute este comando}

-Trae commits especificos de una rama al master o a cualquier otra

-Se ejecuta desde la rama donde quiera traer o incorporar el cambio y no se le tiene que hacer otro commit

-Cherry-pick no es una buena practica pero se puede usar de manera sabia


Amend:

git commit --amend {Cuando hice un commit mal y lo quiero juntar con el commit anterior, quiero que varios cambios esten en el mismo commit pero ya hice un commit con solo un cambio}

-Primero se tiene que hacer el add al cambio que quiero juntar con el ultimo commit y despues ejecuto este comando para asi juntarlos.


Reset y reflog


git reflog {Muestra todos lo cambios que se han hecho, incluso cuando se borra una branch o cualquier cambio, incluyendo resets y todo el historial de cambios}
-Muestra al lado del hash del commit un HEAD(numero)

git reset HEAD(numero) {Hace un reset en la historia de los commits y cambios pero permanece lo que esta en staging}
(Casi nunca se usa)

git reset --hard hashCommit {Con el hash del reflog, me hace un reseteo y me cambia la historia de los commits hasta ese ultimo hashCommit, trae absolutamente cualquier cambio incluyendo archivos borrados}

-Estos resets apareceran en la historia del git reflog, si se quiere se puede devolver a uno de esos errores en caso de cagarla

-Usar git reset es una mala practica y solo debe usarse en casos de emergencia


Tricks


git comando --help {Abre el manual sobre como funciona cada comando}

git config --global alias.arbolito 'log --all --graph --decorate --oneline'
-Crea un alias interno en tu configuracion de git para que asi no se borre siempre que cierras la consola como con los alias normales
- El nombre va despues del alias. y entre "" el comando de git, se ejecuta: git arbolito


git grep palabra {Me muestra en que archivo use esa palabra y como la use}

git grep -n palabra {Me muestra en que linea del archivo use esa palabra}

git grep -c palabra {Muestra cuantas veces use esa palabra en cada archivo}

-Si se quiere buscar una etiqueta en vez de una palabra se escribe con comillas ""


git log -s "palabra" {Muestra dentro del historial de commits cuando y donde se hizo algo relacinado con esa palabra, muestra solo los commits relacionados a esa palabra}

git shortlog {Muestra los commits que ha hecho cada miembro del proyecto, el log por persona}

git shortlog -sn --all {Muestra la cantidad de commits que ha hecho cada miembro del proyecto incluyendo los borrados, si se quiere ver sin lo borrados se le quita el --all}

git shortlog -sn --all --no-merges {Cantidad de commits por miembro sin lo commits de merges}


git blame -c archivo {Muestra que miembro hizo cada linea del archivo, el -c es opcional pero hace que se vea mejor}

git blame archivo -Lnumero1,numero2 {Muestra el blame desde la linea numero1 hasta la linea numero2}
-La L tiene que ser en mayuscula

Github pages y .gitignore


Usar el hosting gratis de git hub:

-creo un repositorio público con el mismo nombre de mi usuario de github, creo un archivo llamado index.html
-voy a los settings de ese repositorio, voy a github pages, y en la parte de source pongo la rama master
-cambió el nombre del repositorio en setting por: simongiraldo.github.io
-otra vez en los settings de git hub pages me sale el link de la página (si no hay un index.html, se tiene que agregar a el link /archivo)


.gitignore:

-Ee crea en archivo en el master con este nombre, ahí se va a poner todos los archivos que hacen que git los ignore, más que todo los binarios como las fotos para evitar problemas con git, ej: *.png *.jpg
-Esto solo son buenas prácticas para no usar archivos binarios en el repositorio local y remoto, asi en github solo aparecerá la carpeta de imágenes y no las imágenes con las extensiones del gitignore
