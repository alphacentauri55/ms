Que es POO


-La programacion orientada a objetos es un paradigma que se basa en objetos coceptuales con diversas herramientas que hacen al codigo mas practico y estructurado. Es un tipo de forma de pensar para plasmar codigo mas manejable. Esto nos ayuda para:

----El codigo muy largo

----Cuando el codigo falla y todo se rompe

----Codigo spaghetti: perdida de control sobre el codigo por controles anidados

POO es la forma de romper el codigo estructurado, las muchas lineas de codigo y las secuencias repetitivas


Principalmente se compone de 4 elementos:

Clases - Propiedades - Metodos - Objetos


Tambien existen en el modelo de POO estos componentes:

Encapsulamiento - Abstraccion - Herencia - Polimorfismo


Lenguajes de programacion:

-Existen varios tipos de lenguajes:

Lenguaje compilado: Genera una compilacion del codigo y luego genera un paquete y despues la maquina virtual del lenguaje lee ese codigo compilado, Ejemplo: java.

Lenguaje interpretado: El navegador web o el servidor va interpretando linea por linea el codigo, Ejemplo: javascript o php

Lenguajes orientados a objetos: Son lenguajes basados en clases que se basan en el concepto de dos entidades distintas: clases e intancias. La mayoria de languajes de este tipo sonn multiparadigma o usan compilacion e interpretacion en distintos momentos, Ejemplo: java, c++.


-La gran mayoria de los lenguajes de programacion usan el paradigma de POO, todos de manera diferente y cada uno con sus particularidades. Esto va a depender del fin del lenguaje y de como este esté construido para ser manipulado.

Objetos, atributos y metodos


-Lo primero que se tiene que hacer es identificar un objeto para resolver un problema

Señales de un objeto:

-Siempre van a tener propiedades y comportamientos

-Su nombre sera un sustantivo

-Pueden ser fisicos o conceptuales


Propiedades:

-Son las caracteristicas de un objeto

-Las propiedades de un objeto tambien se pueden llamar atributos

-Tambien seran sustantivos: Nombre, tamaño, estado, forma, etc...


Metodos:

-Son todas las acciones u operaciones del objeto, comportamientos

-Suelen ser verbos o sustantivo y verbo, siempre con un verbo.


Cada objeto debe de tener sus propios atributos y comportamientos, es muy importante el contexto para saber como definir estos.

-Un objeto es una instancia de la clase. Es decir, el objeto es el resultado de lo que modelamos o los parámetros que dejamos declarados en la clase.


Clases, Abstraccion, Modularidad


Clases: Son los modelos sobre los cuales se contruyen los objetos

Abstraccion: Abstraer los datos o caracteristicas de un objeto para generar un molde

Modularidad: Es un principio que se enfoca en modulos, ayuda a que los elementos vivan separados independientemente y que cumplan el principio de edificacion para gennerar elementos en masa.
Asi cuando ocurra un error este solo afectara a ese modulo y no a toda la edificacion.


Principios de POO gracias a la modularidad:

-Reutilizar
-Evitar colapsos
-Mantenible
-Legibilidad
-Resolucion rapida de problemas


-La Clase sera lo que provoque la modularidad
-Dividir el programa en modulos
-Separar las clases en achivos, es muy buena practica


Herencia


-Crear nuevas clases a partir de otras.

-Reutilizar codigo

-Se establece una relacion padre e hijo

-superclase = padre, subclase = hijo

-Cuando detecto características y comportamientos iguales, entonces significa que debo realizar una abstracción.

-Abstraccion: Cuando las clases hijas heredan los metodos y atributos de la clase padre
